# Glossaire du Projet

## 🎮 Termes Game Design

### **Feature**
Une fonctionnalité complète du jeu (ex: système d'inventaire, combat, quêtes)

### **Mécanique**
Un élément de gameplay interactif (ex: saut, tir, craft)

### **Système**
Un ensemble de mécaniques et règles interconnectées (ex: système de progression)

### **Balancing**
Ajustement des valeurs pour équilibrer la difficulté et le gameplay

### **Game Loop**
Cycle de jeu répétitif core du gameplay

### **UX (User Experience)**
Expérience utilisateur, ressenti global du joueur

### **UI (User Interface)**
Interface utilisateur, éléments visuels d'interaction

## 🏗️ Termes Architecture

### **Agent**
Programme autonome avec une responsabilité spécifique dans le workflow

### **Orchestrateur**
Agent principal qui coordonne tous les autres agents

### **Directeur**
Agent qui supervise une équipe d'agents spécialisés

### **Handoff**
Transfert d'une feature complète d'une équipe à une autre

### **Sync File**
Fichier de communication entre agents

### **READY_FOR_CODE**
État indiquant qu'une feature GDD est complète et peut être implémentée

## 🎯 Termes Unity Spécifiques

### **GameContext**
État actuel du jeu (Menu, Hub, Mission, etc.)

### **UIPanel**
Classe de base pour tous les panneaux d'interface utilisateur

### **Additive Scene**
Scène chargée en complément d'une autre scène active

### **Persistent Scene**
Scène qui reste chargée pendant toute la durée de vie de l'application

### **Canvas Group**
Composant Unity pour gérer la visibilité et l'interactivité d'UI

## 📋 Termes Workflow

### **Spec** (Spécification)
Document détaillé décrivant une feature

### **Integration Report**
Document analysant l'impact d'une feature sur le jeu existant

### **Validation**
Processus de vérification de la qualité et cohérence

### **Refactor**
Restructuration du code sans changer le comportement

### **Blocker**
Élément qui empêche la progression d'une tâche

## 🔄 Status Types

### **In Progress**
En cours de traitement

### **Pending**
En attente d'une action externe

### **Blocked**
Bloqué par un problème

### **Done**
Terminé et validé

## 🎨 Termes UI/UX

### **Fade In/Out**
Animation d'apparition/disparition progressive

### **Tooltip**
Petit popup informatif au survol de souris

### **Drag & Drop**
Interaction de glisser-déposer

### **HUD (Heads-Up Display)**
Interface affichée en surimpression pendant le jeu

### **Modal**
Fenêtre qui bloque l'interaction avec le reste de l'UI

## 📊 Termes Métriques

### **HP (Health Points)**
Points de vie

### **DPS (Damage Per Second)**
Dégâts par seconde

### **Cooldown**
Temps de recharge d'une capacité

### **Stack**
Empilement d'objets identiques

### **Slot**
Emplacement dans un inventaire ou équipement

## 🚀 Termes Techniques

### **Singleton**
Pattern où une classe n'a qu'une seule instance globale

### **Event System**
Système de communication par événements

### **Object Pooling**
Réutilisation d'objets pour optimiser les performances

### **Coroutine**
Fonction Unity pouvant s'exécuter sur plusieurs frames

### **Prefab**
Modèle réutilisable d'objet Unity

---

**Note :** Ce glossaire doit être consulté par tous les agents pour assurer une terminologie cohérente dans tous les documents.
